# Dispatch Telemetry Schema

Event payload schemas for deterministic manager-to-worker dispatch.
These event types replace implicit "publish and hope" patterns with
explicit command/ack/noop flows tied by correlation IDs.

## Event types

### `dispatch.command`

Emitted by the manager when assigning work to a specific agent.

```json
{
  "type": "dispatch.command",
  "correlation_id": "cmd-20260226-001",
  "source": "codex",
  "target": "claude_code",
  "action": "claim_next",
  "task_id": "TASK-abcdef12",
  "timeout_seconds": 120,
  "timestamp": "2026-02-26T00:10:00.000Z"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | string | Yes | Always `"dispatch.command"` |
| `correlation_id` | string | Yes | Unique ID linking command to ack/result/noop |
| `source` | string | Yes | Emitting agent (usually `codex` / manager) |
| `target` | string | Yes | Target agent (`claude_code`, `gemini`) |
| `action` | string | Yes | Command verb: `claim_next`, `work_task`, `sync` |
| `task_id` | string | No | Specific task to claim (if action is `work_task`) |
| `timeout_seconds` | integer | Yes | Seconds before a `dispatch.noop` is emitted |
| `timestamp` | string | Yes | ISO 8601 UTC |

### `dispatch.ack`

Emitted by the worker to confirm receipt of a command.

```json
{
  "type": "dispatch.ack",
  "correlation_id": "cmd-20260226-001",
  "source": "claude_code",
  "task_id": "TASK-abcdef12",
  "status": "accepted",
  "timestamp": "2026-02-26T00:10:05.000Z"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | string | Yes | Always `"dispatch.ack"` |
| `correlation_id` | string | Yes | Matches the originating `dispatch.command` |
| `source` | string | Yes | Acknowledging agent |
| `task_id` | string | No | Task claimed in response to the command |
| `status` | string | Yes | `accepted`, `rejected`, or `busy` |
| `timestamp` | string | Yes | ISO 8601 UTC |

**Status values:**

| Status | Meaning |
|--------|---------|
| `accepted` | Worker claimed the task and will begin work |
| `rejected` | Worker cannot process (no claimable tasks, owner mismatch) |
| `busy` | Worker is still processing a previous task |

### `dispatch.noop`

Emitted by the timeout watcher (watchdog or manager) when neither
an ack nor a result arrives within the command's timeout window.

```json
{
  "type": "dispatch.noop",
  "correlation_id": "cmd-20260226-001",
  "source": "watchdog",
  "target": "claude_code",
  "reason": "ack_timeout",
  "elapsed_seconds": 120,
  "timestamp": "2026-02-26T00:12:00.000Z"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | string | Yes | Always `"dispatch.noop"` |
| `correlation_id` | string | Yes | Matches the originating `dispatch.command` |
| `source` | string | Yes | Agent detecting the timeout |
| `target` | string | Yes | Agent that failed to respond |
| `reason` | string | Yes | `ack_timeout`, `result_timeout`, `agent_disconnected` |
| `elapsed_seconds` | integer | Yes | Seconds since command was emitted |
| `timestamp` | string | Yes | ISO 8601 UTC |

**Reason codes:**

| Reason | Meaning |
|--------|---------|
| `ack_timeout` | No `dispatch.ack` received within timeout |
| `result_timeout` | Ack received but no `worker.result` or `worker.error` |
| `agent_disconnected` | Target agent's heartbeat went stale |

## Correlation ID semantics

- Format: `cmd-YYYYMMDD-NNN` (date + sequence, or UUID)
- Generated by the manager at command emission time
- Must be unique within a session
- All events in a dispatch flow share the same `correlation_id`
- Enables end-to-end tracing: command -> ack -> result/error/noop

## Expected flow

```
Manager                     Worker                  Watchdog
   |                           |                       |
   |--- dispatch.command ----->|                       |
   |                           |--- dispatch.ack ----->|
   |                           |  (working...)         |
   |                           |--- worker.result ---->|
   |                           |                       |
```

**Timeout path (no ack):**

```
Manager                     Worker                  Watchdog
   |                           |                       |
   |--- dispatch.command ----->|                       |
   |                           | (silent)              |
   |                           |           timeout --->|
   |<--- dispatch.noop --------|                       |
```
